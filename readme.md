# Realtime Map

Это веб-приложение для отображения карты с обновлением данных в реальном времени. Построено на Vue 3, Vite и TypeScript.

## Основные возможности

- Отображение интерактивной карты
- Получение и отображение маркеров (объектов) на карте через WebSockets
- Определение геолокации пользователя
- Аутентификация и страница профиля
- Настройки приложения: смена темы (светлая/темная) и языка

## Архитектура проекта (Vertical Slice Architecture)

Проект организован по принципам **Вертикальной слоевой архитектуры (Vertical Slice Architecture, VSA)**. Вместо группировки кода по техническим слоям (например, `controllers`, `services`, `views`), код группируется по функциональным срезам (фичам).

Каждый срез находится в `src/components/02.features/` и является самодостаточным модулем, содержащим все необходимое для работы конкретной фичи:

- **`ui/`**: UI-компоненты (Vue).
- **`model/`**: Логика состояния (Pinia store), типы и composables.
- **`api/`**: Логика для взаимодействия с сервером (если применимо).

### Общие (горизонтальные) слои

Помимо вертикальных срезов, в проекте есть общие слои для переиспользуемого кода:

- **`src/components/01.kit`**: Базовый UI-кит (кнопки, модальные окна и т.д.), который используется всеми фичами.
- **`src/shared`**: Общие утилиты, сервисы (API), плагины и глобальные хранилища (`stores`), которые не относятся к какой-либо конкретной фиче.
- **`src/pages`**: Компоненты-страницы, которые собирают и отображают одну или несколько фич.
- **`src/components/03.layouts`**: Компоненты-макеты для страниц.

Этот подход делает код более модульным, упрощает навигацию и позволяет добавлять или изменять фичи с минимальным воздействием на остальную часть приложения.

## Установка и запуск

Для начала работы с проектом вам понадобится [Node.js](https://nodejs.org/) (версии 18+) и [Bun](https://bun.sh/).

### 1. Клонирование репозитория

Склонируйте репозиторий на свой компьютер:

```sh
git clone https://github.com/hell-soon/RealTime-Map-New.git
cd RealTime-Map-New
```

### 2. Установка зависимостей

Установите все необходимые пакеты с помощью `bun`:

```sh
bun install
```

Эта команда создаст директорию `node_modules`, в которой будут храниться все зависимости проекта.

### 3. Запуск в режиме разработки

Эта команда запустит локальный сервер для разработки (обычно на `http://localhost:5173`).
Сервер поддерживает горячую перезагрузку (Hot-Reload), что позволяет видеть изменения в коде без перезагрузки страницы.

```sh
bun dev
```

### 4. Сборка для production

Для развертывания приложения на сервере его нужно скомпилировать. Эта команда выполнит проверку типов TypeScript, оптимизирует код и соберет все файлы в директорию `dist`.

```sh
bun run build
```

После выполнения этой команды содержимое папки `dist` можно загружать на любой статический хостинг.
